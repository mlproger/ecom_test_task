**запуск и тестирование**

Краткие инструкции по запуску приложения и тестов в проекте.

Требования
- Docker / Docker Compose (рекомендуется)
- Python 3.12 (если запускаете локально)

Настройка
1. Скопируйте шаблон переменных окружения и при необходимости измените значения:

```bash
cp .env.example .env
```

Запуск приложения (Docker)
1. Соберите образ и поднимите сервисы (DB и приложение):

```bash
docker compose up -d db app
```

2. Контейнер `app` запускает `entrypoint`, который ждёт БД и применяет SQL‑миграции из `migrations/`.

3. Приложение будет доступно по умолчанию на порту `8000` (если в `docker-compose.yaml` выставлен проброс).

**Миграции вручную**
Если нужно применить миграции вручную к контейнеру Postgres:

```bash
docker compose exec db psql -U postgres -d postgres -f /workdir/migrations/create_tables.sql
```

Запуск тестов
- Рекомендуемый способ — запуск тестов внутри контейнера, чтобы тесты видели сервис `db` по имени хоста `db` в одной сети:

```bash
docker compose run --rm test
```
